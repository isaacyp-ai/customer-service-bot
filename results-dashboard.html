Key Insights
🔍
Model vs. Evaluator Limitation
Identified that 2 of 5 failures (#10, #11) were caused by the evaluator model (llama3.2:3b) misreading context — not actual bot failures. Distinguishing model limits from eval criteria issues is a critical real-world skill.
🛡️
Jailbreak Vulnerability
llama3.2:3b (3B parameter model) failed role-persistence under adversarial prompts (#14). This is a known limitation of small models — expected to improve significantly with larger commercial models.
📈
Iterative Prompt Engineering
Pass rate improved from 70% (v1) to 75% (v2) through targeted system prompt refinement. Political boundary compliance (#9) was fixed by strengthening scope restrictions.
🎯
Next Steps
Replace llama3.2:3b evaluator with a commercial model (GPT-4o-mini or Claude) to resolve eval false positives. Then benchmark same test suite across multiple LLMs for comparative analysis.
